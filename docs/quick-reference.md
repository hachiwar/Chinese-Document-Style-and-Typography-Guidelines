# 中文排版一页速查表

## 空格

| 规则 | 正确 | 错误 |
| --- | --- | --- |
| 中文与英文之间空一格 | 使用 GitHub 登录 | 使用GitHub登录 |
| 中文与数字之间空一格 | 共 12 个任务 | 共12个任务 |
| 数字与普通单位之间空一格 | 10 GB、8 km | 10GB、8km |
| 百分比、角度、摄氏度不空格 | 15%、90°、37°C | 15 %、90 °、37 °C |
| 中文标点两侧不空格 | 使用 Python，很简单。 | 使用 Python ，很简单。 |
| 不用全角空格 | 中文 文档 | 中文　文档 |

## 标点

| 规则 | 正确 | 错误 |
| --- | --- | --- |
| 中文使用全角标点 | 你好，世界！ | 你好, 世界! |
| 不重复标点 | 完成了！ | 完成了！！！ |
| 中文省略号用六点 | 还没有结束…… | 还没有结束... |
| 破折号前后不空格 | 原则——保持一致 | 原则 —— 保持一致 |
| 并列词语用顿号 | Windows、macOS 和 Linux | Windows，macOS，Linux |
| 多个分句不用顿号 | 先备份，再升级 | 先备份、再升级 |

## 数字与日期

- 正文日期：`2026 年 6 月 25 日`
- 技术日期：`2026-06-25`
- 时间：`14:30` 或 `14 时 30 分`
- 数值范围：`10～20 GB`
- 多位数字：`12,345`
- 版本号：`v2.1.0`
- 百分比：`15%`
- 倍数必须写清“增加了”还是“增加到”

## 英文与名称

- 使用官方大小写：GitHub、JavaScript、MySQL、macOS、iPhone。
- 不自造缩写：写“TypeScript”，不写“Ts”。
- 不常见缩略语首次出现时解释全称。
- 完整英文句子使用英文标点。
- URL、代码、文件路径、命令和型号保持原样。

## 标题与段落

- 标题禁止跳级，通常不超过四级。
- 操作型标题使用“动词 + 对象”，例如“配置数据库”。
- 标题末尾通常不加句号。
- 一段只讲一个主题，中心句优先放在段首。
- 句子建议不超过 100 字，段落建议为 50～200 字。
- 避免“一逗到底”，复杂信息改用列表或表格。

## 列表与表格

- 顺序重要时用有序列表，否则用无序列表。
- 同一列表保持句式、粒度和标点一致。
- 列表嵌套建议不超过三级。
- 表格必须有表头，空值必须说明含义。
- 不在单元格中写长段落，不使用“同上”。

## 链接与代码

- 链接文字描述目标内容，不写“点击这里”。
- 行内命令、参数、文件名、路径和字段用反引号。
- 代码块标注语言，命令和输出分开。
- 危险命令说明影响、备份和回滚方式。

## 发布前十问

1. 事实、版本和日期准确吗？
2. 专有名词大小写正确吗？
3. 中英文与数字之间留空格了吗？
4. 中文标点使用全角了吗？
5. 引号、括号和日期格式统一吗？
6. 标题层级连续吗？
7. 长句和长段落拆分了吗？
8. 链接能打开且文字有意义吗？
9. 代码和步骤经过验证吗？
10. 是否由另一位读者审校过？
