# 发布前审校清单

## 1. 内容正确性

- [ ] 文档目标和目标读者明确。
- [ ] 事实、数据、日期、版本和名称准确。
- [ ] 相对时间已经补充具体日期。
- [ ] 引用、图片和第三方内容已经注明来源。
- [ ] 链接可访问，且没有误导性跳转。
- [ ] 示例、代码和命令已经验证，或已明确标注为伪代码。

## 2. 语言与术语

- [ ] 全文使用简体中文或繁体中文，没有混用。
- [ ] 专有名词采用官方拼写和大小写。
- [ ] 不常见缩略语在首次出现时已经解释。
- [ ] 同一概念使用同一术语。
- [ ] “你”与“您”没有混用。
- [ ] 代词指向明确，没有因“其”“该”“此”产生歧义。
- [ ] 没有无依据的“最佳”“最先进”“显著”等绝对化表述。
- [ ] 没有行话、黑话、歧视性称呼或过度网络化表达。

## 3. 空格与字符

- [ ] 中文与英文之间有一个半角空格。
- [ ] 中文与阿拉伯数字之间有一个半角空格。
- [ ] 数字与普通单位之间有一个半角空格。
- [ ] 百分比、角度和摄氏度前没有空格。
- [ ] 中文标点两侧没有多余空格。
- [ ] 没有全角空格或无意义的连续空格。
- [ ] 数字和英文字母使用半角字符。

## 4. 标点

- [ ] 中文语境使用全角中文标点。
- [ ] 引号形式在全文一致。
- [ ] 括号形式在全文一致。
- [ ] 没有重复使用问号或叹号。
- [ ] 中文省略号使用“……”。
- [ ] 破折号使用“——”，前后不加空格。
- [ ] 顿号用于词语并列，逗号或分号用于分句。
- [ ] 数值范围使用统一连接号。

## 5. 结构与排版

- [ ] 标题准确概括内容。
- [ ] 标题层级连续，没有跳级。
- [ ] 没有孤立的单个子标题。
- [ ] 一个段落只讨论一个主题。
- [ ] 长句已经拆分，没有“一逗到底”。
- [ ] 列表项句式、粒度和标点一致。
- [ ] 有序列表只用于顺序重要的内容。
- [ ] 表格有表头，空值含义明确。
- [ ] 链接文字能够描述目标内容。

## 6. 技术内容

- [ ] 行内代码、命令、参数、文件名和字段使用反引号。
- [ ] 代码块标注语言。
- [ ] 命令与输出分开。
- [ ] 占位符已经解释替换方法。
- [ ] 步骤包含前置条件和预期结果。
- [ ] 危险操作说明后果、备份和回滚方法。
- [ ] 错误信息按原文引用，并提供诊断和解决方法。

## 7. 图片与无障碍

- [ ] 图片清晰，且没有泄露隐私、账号或密钥。
- [ ] 信息型图片具有有效替代文本。
- [ ] 图注说明图片表达的信息。
- [ ] 外部图片注明来源和许可。
- [ ] 信息不只依赖颜色传达。

## 8. 最终确认

- [ ] 已运行拼写、Markdown 或风格检查工具。
- [ ] 已在目标平台预览排版。
- [ ] 已检查移动端或窄屏阅读效果。
- [ ] 已由至少一位非作者审校。
- [ ] 审校者可以仅依靠本文档完成目标任务。
