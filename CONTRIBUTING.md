# 贡献指南

本项目接受对规范内容、示例、依据和项目结构的修订。

## 1. 可接受的贡献

- 修复错别字、病句、失效链接和格式问题；
- 增加具有代表性的正误示例；
- 补充国家标准、行业标准或权威资料；
- 指出规则冲突、适用边界和地区差异；
- 增加无障碍、国际化或特定内容类型的规范；
- 改进自动化检查方案。

## 2. 新增规则的基本要求

新增或修改规则时，应尽量说明：

1. 规则所解决的问题；
2. 规则的适用范围；
3. 规则属于强制、推荐还是可选；
4. 规则的例外；
5. 至少一组正确和错误示例；
6. 标准、官方文档或其他可靠依据。

个人审美不应直接规定为强制规则。如果某一问题存在多种合理方案，应将其列为团队可选规则，并强调一致性要求。

## 3. 文案要求

- 使用简体中文和全角中文标点。
- 中文与英文、数字之间按本项目规范使用空格。
- 专有名词使用官方拼写。
- 不大段复制第三方文章。
- 引用第三方观点时注明来源。
- 每个 Pull Request 聚焦一个主题。

## 4. 本地检查

提交前至少检查：

- Markdown 标题没有跳级；
- 相对链接可以正常打开；
- 文件名符合小写短横线规范；
- 没有明显的中英文空格和标点错误；
- README 导航与实际文件一致。

可以使用以下工具辅助检查：

- [Markdownlint](https://github.com/DavidAnson/markdownlint)
- [Vale](https://vale.sh/)
- [AutoCorrect](https://github.com/huacnlee/autocorrect)
- [pangu.js](https://github.com/vinta/pangu.js)

## 5. 版权

提交贡献即表示贡献者有权提交相关内容，并同意将原创贡献按本项目的 MIT License 发布。第三方内容必须遵循原始许可和署名要求。
